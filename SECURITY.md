## Reporting a Vulnerability

We take security seriously at Transilience AI. If you discover a security vulnerability in any of our repositories, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

1. **Email:** Send details to [security@transilience.ai](mailto:security@transilience.ai)
2. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Affected repository and version
   - Potential impact assessment
   - Any suggested fix (optional)

### What to Expect

- **Acknowledgment** within 48 hours of your report
- **Initial assessment** within 5 business days
- **Regular updates** on the status of your report
- **Credit** in the fix announcement (unless you prefer to remain anonymous)

### Scope

This policy applies to all public repositories under the [transilienceai](https://github.com/transilienceai) GitHub organization.

### Out of Scope

- Vulnerabilities in third-party dependencies (report these upstream, but let us know so we can update)
- Social engineering attacks against Transilience AI staff
- Denial of service attacks against our infrastructure

## Security Best Practices

When using our tools, especially [communitytools](https://github.com/transilienceai/communitytools), ensure you:

- Only test against systems you own or have explicit written authorization to test
- Never use security tools against production systems without proper approval
- Follow responsible disclosure practices for any vulnerabilities you discover
